MSPGCC 4.5.3 LTS20110716 Release
Long Term Support (LTS) (DEPRECATED)
Replaced by MSPGCC 4.6.3 (LTS)

This release is deprecated. You should be using MSPGCC 4.6.3 (LTS). See README_46 for details.

The 4.5.3-LTS20110716 repository contains the following packages for i386:

There are no plans to update it nor to provide amd64 packages.

  • msp430-45: 20120427
  • msp430-binutils-45: 2.21.1-LTS20120406
  • msp430-gcc-45: 4.6.3-LTS20120406
  • msp430-gdb-45: 7.2-LTS20120406
  • msp430-libc-45: 20120224-LTS20120406
  • msp430mcu-45: 20120406-LTS20120406

To use this repository you need to:

  1. Add the following line to /etc/apt/sources.list.d/tinyprod-debian.list: 
      deb http://tinyprod.net/repos/debian msp430-45 main
  deb http://tinyprod.net/repos/debian squeeze main

    
  $ cd /etc/apt/sources.list.d
  $ sudo echo "deb http://tinyprod.net/repos/debian msp430-45 main" >> tinyprod-debian.list
  $ sudo echo "deb http://tinyprod.net/repos/debian squeeze main" >> tinyprod-debian.list
  2. Install the new packages: 
      $ sudo apt-get update
  $ sudo apt-get install msp430-45 nesc tinyos-tools

If you want to uninstall the packages you can do it like this:

  $ sudo apt-get autoremove --purge msp430-45 nesc tinyos-tools
Note:

  1. Other packages from the main tinyprod debian repository (squeeze) are needed to make a functioning nesc development sytem. In particular, you will need nesc and tinyos-tools. These packages can be found in the squeeze repository. nesc and tinyos-tools install into the standard /usr hierarchy and should be found by searching /usr/bin. You will have to add /opt/msp430-45/bin to pick up the msp430-45 binaries.
  2. The msp430-45 tools install into /opt/msp430-45 to avoid conflict with standard toolchain installs.

    Eventually, released msp430 toolchains from standard distribution archives will be used. These will install into the standard /usr hierarchy. Using /opt/msp430-45 will avoid conflicting with these locations.

    In addition, seperate install paths for different toolchains will allow the default and experimental mspgcc compilers to exist side by side on the same system.

    Selection of the operational toolchain is done via the PATH environment variable.
